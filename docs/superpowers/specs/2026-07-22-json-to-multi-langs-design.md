# JSON → 多语言转换器设计

**日期**: 2026-07-22
**分支**: `feat/json-to-multi-langs`
**状态**: 设计已确认，待实现

## 背景

`tools` 项目已有三个 JSON 转换工具：

- `json-to-go` — JSON 样本 → Go 结构体
- `json-to-go-map` — JSON → Go `map[string]interface{}` 字面量
- `json-to-ts` — JSON → TypeScript 类型定义

三者均基于 `quicktype-core`，UI 共用同款 CodeMirror 双栏布局。用户希望新增更多主流语言的转换工具，拆为独立工具（独立路由、侧栏入口、源码文件）。

## 目标

新增 5 个独立工具：

| 工具名 | 路由 | 目标语言 |
|---|---|---|
| JSON → Python | `/json-to-python` | Python |
| JSON → Rust | `/json-to-rust` | Rust |
| JSON → Java | `/json-to-java` | Java |
| JSON → C++ | `/json-to-cpp` | C++ |
| JSON → C# | `/json-to-csharp` | C# |

## 非目标

- **不重构**现有 `json-to-go` / `json-to-go-map` / `json-to-ts` 的源码。它们的测试保持不动。
- **不抽公共组件**。5 个新工具各自独立 copy CodeMirror 双栏模板。重复约 200 行模板代码是可接受代价，换取零回归风险与最小改动面（CLAUDE.md §3 surgical changes）。
- 不支持 JSON Schema 输入，只接受 JSON 样本。
- 不做「单页面多语言切换」UI，每个语言是独立路由。

## 架构

### 目录结构

每个新工具独立目录，与现有 `json-to-go` 结构完全对齐：

```
src/tools/
├── json-to-python/
│   ├── JsonToPython.vue
│   ├── json-to-python.js
│   └── json-to-python.test.js
├── json-to-rust/        (同上三件套)
├── json-to-java/        (同上)
├── json-to-cpp/         (同上)
└── json-to-csharp/      (同上)
```

### 现有代码不动

`json-to-go`、`json-to-go-map`、`json-to-ts` 的 `.vue` / `.js` / `.test.js` 全部保留原样。

## 各语言选项与渲染行为

所有工具共用 `typeName` 输入框（顶层类型名，默认 `AutoGenerated`，非法标识符时 fallback + warning，机制与 `json-to-go` 一致）。所有工具默认 `just-types: true`，与现有 Go/TS 保持一致——只输出类型声明，不输出 JSON 反序列化 runtime 代码。

### JSON → Python

- 路由: `/json-to-python`
- CodeMirror: `@codemirror/lang-python`（已装）
- 顶层类型名: `typeName`，标识符正则 `^[A-Za-z_][A-Za-z0-9_]*$`
- Checkbox 选项:
  1. **仅类型（无 runtime）** — `just-types: true`（默认勾选）
  2. **所有属性可选** — `allPropertiesOptional: true`（顶层 option，让单样本字段渲染为 `Optional[T]`）
  3. **Pydantic BaseModel** — `pydantic-base-model: true`
- quicktype `lang`: `'python'`

### JSON → Rust

- 路由: `/json-to-rust`
- CodeMirror: `@codemirror/lang-rust`（已装）
- 顶层类型名: `typeName`
- Checkbox 选项:
  1. **仅类型** — `just-types: true`（默认勾选）
  2. **Derive Debug** — `derive-debug: true`（quicktype 默认 true，UI 默认勾选）
  3. **Derive Clone** — `derive-clone: true`（quicktype 默认 true，UI 默认勾选）
  4. **跳过序列化 None** — `skip-serializing-none: true`
- quicktype `lang`: `'rust'`

### JSON → Java

- 路由: `/json-to-java`
- CodeMirror: `@codemirror/lang-java`（已装）
- 顶层类型名: `typeName`
- Checkbox 选项:
  1. **仅类型（POJO，无 Jackson 注解）** — `just-types: true`（默认勾选）
  2. **使用 Lombok** — `lombok: true`
  3. **数组用 `List<T>` 而非 `T[]`** — `array-type=list`（默认勾选，对应 quicktype 默认）
- quicktype `lang`: `'java'`

### JSON → C++

- 路由: `/json-to-cpp`
- CodeMirror: `@codemirror/lang-cpp`（已装）
- 顶层类型名: `typeName`
- Checkbox / 输入框:
  1. **仅类型** — `just-types: true`（默认勾选）
  2. **namespace** 输入框 — `namespace`，默认 `quicktype`
- quicktype `lang`: `'cpp'`

### JSON → C#

- 路由: `/json-to-csharp`
- CodeMirror: **需新增 `@replit/codemirror-lang-csharp` 依赖**（CodeMirror 6 官方未发布 C# 语言包，使用 Replit 维护的社区包，API 与官方 `@codemirror/lang-*` 一致，导出 `csharp()`）
- 顶层类型名: `typeName`
- Checkbox / 输入框:
  1. **仅类型** — `just-types: true`（默认勾选）
  2. **namespace** 输入框 — `namespace`，默认 `QuickType`
  3. **数组用 `List<T>` 而非 `T[]`** — `array-type=list`（默认勾选）
- quicktype `lang`: `'csharp'`

## 文件模板

### `json-to-<lang>.js`（每个约 40-60 行）

Copy 自 `src/tools/json-to-go/json-to-go.js`，改动点：

1. 函数名 `jsonToGo` → `jsonTo<Lang>`
2. `jsonInputForTargetLanguage('go')` → 对应语言 id
3. `quicktype({ lang: 'go', ... })` → 对应语言 id
4. options 解构与 `rendererOptions` 按 §各语言选项 填
5. 标识符正则按语言调整（Python/Rust/Java/C++/C# 基本都可用 `^[A-Za-z_][A-Za-z0-9_]*$`）
6. warning 文案里的语言名替换

返回值结构不变：`{ ok: true, code, warning? } | { ok: false, error }`。

### `JsonTo<Lang>.vue`（每个约 200-230 行）

Copy 自 `src/tools/json-to-go/JsonToGo.vue`，改动点：

1. `<h1>` 标题文案
2. toolbar 里的 checkbox / 输入框，按 §各语言选项 配置
3. `import` 的 CodeMirror 语言包（`@codemirror/lang-python` 等）
4. `createEditors()` 里 output editor 的 lang 扩展（如 `python()` / `rust()` / `java()` / `cpp()` / `csharp()`）
5. `import { jsonTo<Lang> } from './json-to-<lang>.js'`
6. `generate()` 里传给 `jsonTo<Lang>` 的参数

CodeMirror 双栏模板（editor 生命周期 / debounce / copy / theme 切换 / 清空）原样 copy，不改动。

### `json-to-<lang>.test.js`（每个约 80-120 行）

Copy 自 `src/tools/json-to-go/json-to-go.test.js`，改动点：

1. `import { jsonTo<Lang> } from './json-to-<lang>.js'`
2. 类型声明关键字断言按语言调整：
   - Python: `class \w+` 或 `class \w+\(`（pydantic）
   - Rust: `struct \w+`
   - Java: `class \w+`
   - C++: `struct \w+`
   - C#: `class \w+` 或 `struct \w+`

测试用例对齐现有 Go 测试：

- empty input → `ok: true`, `code: ''`
- invalid JSON → `ok: false`, `error` 匹配 `/JSON/`
- flat object → `ok: true`，输出含类型声明关键字
- nested object → 至少 2 个类型声明
- 默认 `just-types: true` → 输出不含 runtime 反序列化代码（语言相关断言）
- 自定义 `typeName` → 输出命中
- 非法 `typeName` → fallback `AutoGenerated` + `warning` 匹配 `/合法.*标识符/`
- 合法 `typeName` → `warning` 为 `undefined`
- 空 `typeName` → `warning` 为 `undefined`
- 顶层 array / null 字段 / 混合类型数组 → `ok: true`
- 至少一个工具专属选项的行为断言（如 Python `pydantic: true` → 输出含 `BaseModel`）

## 注册步骤

按 CLAUDE.md「How to Add a New Tool」三步流程，每个工具：

1. 创建 `src/tools/json-to-<lang>/` 目录 + 三件套文件
2. `src/router.js` 的 `components` 映射里追加：
   ```js
   '/json-to-<lang>': () => import('./tools/json-to-<lang>/JsonTo<Lang>.vue'),
   ```
3. `src/tools.js` 在「编码转换」分组末尾（`json-to-ts` 之后）追加入口：
   ```js
   {
     name: 'JSON → Python',
     path: '/json-to-python',
     desc: '将 JSON 样本转换为 Python 类型定义',
     icon: CodeBracketSquareIcon,
   },
   // ...Rust/Java/C++/C# 同样
   ```
   `CodeBracketSquareIcon` 已在 `tools.js` import，无需新加 icon。

## 风险与回退

- **quicktype 行为差异**：某些语言的 `just-types` 在单样本下可能不产生 `omitempty` 等效果，与 Go 一致——通过 `allPropertiesOptional` 顶层 option 解决。实现时若发现某语言选项不生效，在 `.js` 文件顶部注释中记录实际行为（参照 `json-to-go.js` 的注释风格）。
- **CodeMirror C# 包**：官方 `@codemirror/lang-csharp` 不存在，改用 `@replit/codemirror-lang-csharp`。若该社区包与当前 CodeMirror 6 版本不兼容，C# 工具可降级为不挂语言扩展（用默认高亮），不阻塞其他工具。
- **回退**：所有改动在新分支 `feat/json-to-multi-langs` 上，不合并即无影响。
