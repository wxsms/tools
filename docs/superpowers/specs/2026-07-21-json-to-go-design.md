# JSON → Go Struct 工具设计

## 背景

wxsm's Kit 当前没有"JSON 样本 → 强类型代码"类工具。quicktype（glideapps/quicktype）能在浏览器里通过 `quicktype-core` npm 包直接生成 Go 结构体，本项目希望加一个面向 Go 开发者的转换工具，沿用项目既有的"双 textarea 转换器"交互范式（参考 `src/tools/url-encode/`）。

## 目标

用户粘贴一份 JSON 样本，工具实时输出对应的 Go 结构体定义，可一键复制。

## 范围

**In scope**

- 接受一份 JSON 样本作为输入（不接受 JSON Schema，不从 URL 拉取）。
- 通过 `quicktype-core` 生成 Go 代码。
- 一个 `omitempty` 开关，控制生成的 Go tag 是否带 `omitempty`。
- 输入 parse 失败 / 生成失败时在界面显示错误信息。
- 输出框只读、可复制；提供清空按钮。
- 输入框预填一段示例 JSON。

**Out of scope**

- JSON Schema 输入、从 URL 拉取 JSON。
- 可配置 package 名（生成的代码不写 package 行或在工具内部固定）。
- 多余的"一键复制"按钮（textarea 右下角已有复制按钮即可满足）。
- Go → JSON 反向转换。
- CodeMirror 编辑器（保持与 `url-encode` 一致的普通 textarea）。
- "嵌套对象是否拆为独立 struct"开关 —— quicktype-core 的 Go 渲染器默认即拆为独立 struct，无 toggle 对应的渲染选项，加一个装饰性开关会误导用户，故不设。

## 架构

### 文件布局

```
src/tools/json-to-go/
├── JsonToGo.vue          # 视图，镜像 url-encode 的双 textarea 布局
├── json-to-go.js         # 纯模块，封装 quicktype-core
└── json-to-go.test.js    # Vitest 单元测试，fixture based
```

### 三处接线（CLAUDE.md "How to Add a New Tool" 必须步骤）

1. `src/router.js`：增加 `'/json-to-go': lazy(() => import('./tools/json-to-go/JsonToGo.vue'))`。
2. `src/routes.js`：追加 `{ path: '/json-to-go', meta: { title: 'JSON → Go', description: '将 JSON 样本转换为 Go 结构体定义，支持 omitempty' } }`。
3. `src/tools.js`：在"编码转换"分组追加条目，icon 复用已 import 的 `CodeBracketSquareIcon`。

### 依赖

`package.json` 增加运行时依赖 `quicktype-core`。该包可在浏览器 ESM 环境直接 import，无完整 TypeScript 类型声明，本项目是 JS 项目因此无影响。

## 组件设计

### `json-to-go.js`（纯模块）

公开 API：

```js
export async function jsonToGo(jsonString, { omitempty = true } = {})
  // → { ok: true, code: string } | { ok: false, error: string }
```

行为：

1. 空字符串输入：直接返回 `{ ok: true, code: '' }`。
2. 先 `JSON.parse(jsonString)` 做一次解析；失败则返回 `{ ok: false, error: 'JSON 解析失败：' + e.message }`。提前 parse 是为了拿到清晰的语法错误信息，不依赖 quicktype 内部的报错路径。
3. 调用 `quicktype-core` 的 `quicktype({ source: { kind: 'json', samples: [jsonString] }, targetLanguage: 'go', rendererOptions: { ... } })`，渲染器选项根据 `omitempty` 切换。**实现时需对照 `quicktype-core` Go renderer 的 option 名最终确认**（候选为 `just-types` + `just-types-and-omitempty`，若名称有出入以实测为准）。
4. 整个 quicktype 调用包在 try/catch 里；失败时返回 `{ ok: false, error: '生成失败：' + e.message }`。
5. 成功则返回 `{ ok: true, code: result.lines.join('\n') }`（quicktype-core 的返回结构以实测为准）。

返回 discriminated union（`ok` 字段），让 Vue 侧无需 try/catch 包裹错误路径，与 `url-encode` 的错误内联展示风格一致。

### `JsonToGo.vue`（视图）

布局镜像 `src/tools/url-encode/UrlEncode.vue`：

```
┌──────────────────────────────────────┐
│ JSON → Go 转换                       │
│                                      │
│ [✓ omitempty]   (单个 toggle)        │
│                                      │
│ ┌──────────────────────┐             │
│ │ JSON 输入            │  textarea   │
│ └──────────────────────┘             │
│           ⇅ ArrowsUpDownIcon         │
│ ┌──────────────────────┐             │
│ │ Go 输出（只读）      │  textarea   │
│ └──────────────────────┘             │
│ [error line if any]                  │
│                         [清空]       │
└──────────────────────────────────────┘
```

State：

- `input`：默认预填一段示例 JSON（含嵌套对象、数组、多种标量类型，使输出能展示典型结构）。
- `output`：默认 `''`，挂载后由初次生成填充。
- `error`：默认 `''`。
- `omitempty`：默认 `true`。
- `inputCopied` / `outputCopied`：复制按钮的瞬时反馈状态。

Behavior：

- `onInputChange()`：debounce ~300ms（quicktype-core 异步，避免每次按键触发）。清空 `error`，调用 `jsonToGo(input.value, { omitempty: omitempty.value })`，按返回结果写入 `output` 或 `error`。
- toggle `omitempty`：立刻重跑（不 debounce），因为只是根据现有 input 重新渲染。
- 输入为空：`output = ''`，不显示错误。
- 输出 textarea 设为 `readonly`（单向转换，避免用户编辑后被下次生成覆盖造成困惑）。
- 复制按钮：右下角 ghost btn-square，`ClipboardDocumentIcon` / `CheckIcon` 切换，与 `url-encode` 完全一致。
- 清空按钮：重置 `input`、`output`、`error`。
- 不设 `meta.wide`，沿用默认 `max-w-4xl`。

### 错误处理

三种失败模式都通过同一 `error` ref 显示一行：

1. JSON 语法错误：`JSON 解析失败：<原因>`。
2. quicktype 运行时错误：`生成失败：<原因>`。
3. 空输入：不显示错误，`output` 清空。

无重试、无兜底，用户根据错误信息调整 JSON。

## 测试策略

`json-to-go.test.js` 使用 Vitest，fixture based，对**子串**断言而非整文本对比（quicktype 输出格式可能随版本微调，整文本快照脆弱）：

- 标量字段对象 → 输出包含 `type AutoGenerated struct`、包含 Go 基础类型字面量。
- 嵌套对象 → 输出包含第二个 `type ... struct`。
- 对象数组 → 输出包含 `[]...` 形式的字段类型。
- `omitempty: true` → 输出包含 `omitempty`。
- `omitempty: false` → 输出不包含 `omitempty`。
- 非法 JSON → 返回 `{ ok: false, error: /JSON/ }`。
- 空字符串 → 返回 `{ ok: true, code: '' }`。

不为 `.vue` 写组件测试 —— 组件是纯模块的薄壳，且项目既有 `url-encode` 也只为纯函数写单测，沿用同一约定。

## 实现 note

- `quicktype-core` 的具体 import 形态、`quicktype()` 入参结构、Go renderer 的 option 名以实现时实测为准；本设计的 API 形态（`jsonToGo(json, { omitempty }) → { ok, code | error }`）固定不变，内部实现细节允许根据实际 API 调整。
- 若 `quicktype-core` 在 vitest 下不易运行（例如依赖 Node 内置模块），降级为 fixture-based 烟测：在测试里 stub `quicktype-core` 的输出，只验证 `json-to-go.js` 的输入校验、空输入短路、错误包裹逻辑。这是 fallback，首选仍是跑真实的 quicktype。
