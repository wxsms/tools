# CLI 格式转换工具设计

**日期**:2026-06-24
**工具名**:命令行格式转换
**路径**:`/cli-format`
**分组**:文本处理
**图标**:`CommandLineIcon` (`@heroicons/vue/24/outline`)

## 目的

把命令行字符串在单行风格与多行风格之间互转,便于在"易读(多行)"和"可直接执行(单行)"两种形态间切换。典型场景:

- 从文档/终端复制多行带 `\` 续行的命令,拼成一行去执行
- 把一长行命令拆开,看清楚每个参数

## 范围

- **仅支持 Unix shell 风格**(`\` 续行、`'...'` / `"..."` 引号、反斜杠转义)
- 不支持 Windows CMD(`^` 续行、`%VAR%`)
- 不展开变量、不解析管道/重定向(整个输入当一个命令及参数处理)

## 非目标

- 不做 shell 语法高亮
- 不做命令补全 / 校验命令是否存在
- 不重写引号(原样保留)
- 不识别 boolean 选项(避免猜错)

## 布局

参考 `src/views/UrlEncode.vue` 的双向转换模式:

```
┌─────────────────────────────────────────────┐
│ 命令行格式转换 / 解码                        │
│                                             │
│ [方向: 单行→多行] [方向: 多行→单行]          │
│                                             │
│ [续行符: 带\ / 无]  [缩进: 0 / 2 / 4]        │
│ (配置仅影响 单行→多行 输出)                  │
│                                             │
│ ┌─────────────────────────┐                 │
│ │ 输入框                   │  [复制]         │
│ └─────────────────────────┘                 │
│              ↕ (双向自动同步)                │
│ ┌─────────────────────────┐                 │
│ │ 输出框                   │  [复制]         │
│ └─────────────────────────┘                 │
│ [清空]                       [错误提示]      │
└─────────────────────────────────────────────┘
```

- 顶部 toolbar:配置开关(续行符、缩进)。**无方向切换** —— 双向自动同步,任一框修改都触发对应方向转换
- 两个 textarea(单行 / 多行),双向自动同步:
  - 单行框 input → 用 `toMultiLine` 重算多行框
  - 多行框 input → 用 `toSingleLine` 重算单行框
  - 选项(续行符/缩进)变化时,以单行框为源重算多行框
- 中间箭头图标(纯视觉提示,不可点击)
- 每个框右下角复制按钮
- 底部清空按钮 + 错误提示区

## 配置开关

| 开关 | 选项 | 默认 | 影响 |
|------|------|------|------|
| 续行符 | 带 `\` / 无 | 带 `\` | 仅 单行→多行 |
| 缩进 | 0 / 2 / 4 空格 | 2 | 仅 单行→多行 |

## 转换算法

### Token 解析(两个方向共用)

把输入字符串解析成 token 数组,规则:

1. **续行处理**:行末单独的 `\` + 换行 → 吃掉换行和下一行前导空白,继续拼接
2. **空白拆分**:非引号内的空白(空格 / Tab / 换行)是 token 分隔符
3. **单引号** `'...'`:内容原样,直到下一个 `'`(不支持转义)
4. **双引号** `"..."`:内容直到下一个未转义的 `"`,反斜杠仅转义 `\"` `\\` `\$` `` ` ``
5. **反斜杠转义**(非引号内):`\X` 当作字面 `X`(常见 `\ ` `\\` `\"` `\$`)
6. **`=` 不拆分**:`--output=/path` / `-o=/path` 作为单个 token,不按 `=` 拆分(等号是 token 内部字符,不是分隔符)
7. **引号原样保留**:解析出 token 的同时,记录其原始写法(是否带引号、哪种引号),输出时按原写法还原

### 多行 → 单行

1. 用 Token 解析拆出 tokens
2. 用单空格拼接所有 token(按原始写法)
3. 结果即为单行命令

### 单行 → 多行

1. 用 Token 解析拆出 tokens
2. 第一个 token 单独一行(命令本体),不加缩进,不加续行符
3. 后续 token 按"flag + 值"启发式分组:
   - 当前 token 以 `-` 开头 → 视为 flag
   - 下一个 token 不以 `-` 开头 → 作为该 flag 的值,同行
   - 否则当前 flag 单独成行
   - 不以 `-` 开头的 token(位置参数)单独成行
4. 每行加配置的缩进
5. 若续行符开关为"带 `\`":除最后一行外,每行末尾追加 ` \`
6. 拼接成多行字符串

**边界**:值为 `-` (如 `--output -`)会被当作下一个 flag,单独成行。这是已知取舍(不猜 boolean),可接受。

## 容错

- 引号未闭合:显示错误 `引号未闭合:缺少 '...' / "..."`,输出框清空
- 输入为空或纯空白:输出框清空,无错误
- 解析过程中任何异常:捕获,显示 `解析失败: <message>`,输出框清空
- 错误不破坏输入框内容

## 测试要点

- 单/双引号内带空格的值:`--name "hello world"` ↔ 多行
- 反斜杠续行 + 引号混合:多行带 `\` 续行,引号跨行
- Windows 路径不被误判:`C:\Users\foo` 中的 `\U` `\f` 不当转义
- 空输入、纯空白
- 单 token(无参数,仅命令名)
- 位置参数:`command foo bar`
- 多个 flag 紧挨:`-a -b -c`
- 长选项带 `=`:`--output=/path` (应作为单个 token,不拆分)
- 末尾带 `\` 但没有下一行(非法续行)

## 验收标准

1. 两个 textarea 双向自动同步(无方向按钮),任一框 input 触发对应方向转换
2. 续行符 / 缩进开关正确影响 单行→多行 输出;选项变化时以单行框为源重算多行框
3. Unix 引号与转义正确处理,原引号保留
4. 非法输入显示错误,不崩溃
5. `npm run lint` 通过
6. `npm run test` 通过
7. 侧边栏"文本处理"组出现新条目,可正常导航
