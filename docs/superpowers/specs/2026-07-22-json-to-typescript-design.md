# JSON → TypeScript 工具设计

日期：2026-07-22
状态：已批准，待实现

## 目标

新增一个独立工具 `JSON → TypeScript`，将 JSON 样本转换为 TypeScript 类型定义。结构对照现有 `json-to-go` 工具，复用 `quicktype-core` 作为后端。

## 非目标

- 不做多目标语言切换（保持独立工具形态）
- 不重构 `json-to-go` 抽公共组件
- 不支持 JSON Schema / TypeScript 类型作为输入，仅支持 JSON 样本

├── json-to-ts.js         # 纯函数，调 quicktype-core 生成 TS

## 纯函数 `jsonToTs`

签名：

```js
async function jsonToTs(jsonString, {
  allOptional = false,
  preferTypes = false,
  typeName = 'AutoGenerated',
} = {}) -> { ok: true, code: string, warning?: string } | { ok: false, error: string }
```

行为：

1. 空输入（空字符串或仅空白）→ `{ ok: true, code: '' }`
2. `JSON.parse` 失败 → `{ ok: false, error: 'JSON 解析失败：' + e.message }`
3. `typeName` 校验：合法 TS 标识符正则 `^[A-Za-z_$][A-Za-z0-9_$]*$`
   - 空字符串视为合法，使用默认 `AutoGenerated`，无 warning
   - 非法 → 回退 `AutoGenerated`，返回 warning `结构体名 "<name>" 不是合法 TypeScript 标识符`
4. 调 `quicktype-core`：
   ```js
   const jsonInput = jsonInputForTargetLanguage('typescript')
   await jsonInput.addSource({ name: safeName, samples: [JSON.stringify(parsed)] })
   const inputData = new InputData()
   inputData.addInput(jsonInput)

   const rendererOptions = { 'just-types': true }
   if (preferTypes) rendererOptions['prefer-types'] = true
   const extraOptions = {}
   if (allOptional) extraOptions.allPropertiesOptional = true

   const result = await quicktype({
     lang: 'typescript',
     inputData,
     rendererOptions,
     ...extraOptions,
   })
   return { ok: true, code: result.lines.join('\n'), warning }
   ```
5. 生成失败 → `{ ok: false, error: '生成失败：' + e.message }`

renderer option 说明（已通过查看 `node_modules/quicktype-core` 源码确认）：
- `just-types: true` —— 只输出类型声明，不输出 runtime 解析代码
- `prefer-types: true` —— 输出 `type Foo = { ... }`；不设置或 false 输出 `interface Foo { ... }`
- `allPropertiesOptional: true`（quicktype 顶层 option，非 renderer option）—— 让所有字段渲染为可选 `field?: T`

## Vue 视图 `JsonToTs.vue`

布局对照 `JsonToGo.vue`，差异仅在：

- 工具栏三个控件：
  1. 顶层类型名输入框（`v-model="typeName"`，placeholder `AutoGenerated`）
  2. "所有属性可选" checkbox（`v-model="allOptional"`）
  3. "用 type 别名" checkbox（`v-model="preferTypes"`）
- 输入编辑器：`@codemirror/lang-json` 高亮
- 输出编辑器：`@codemirror/lang-typescript` 高亮，只读
- 默认输入示例：
  ```js
  {
    name: 'wxsm',
    age: 18,
    active: true,
    tags: ['dev', 'ts'],
    address: { city: 'BJ', zip: '100000' },
  }
  ```
- 复制按钮、清空按钮、错误/警告区、debounce 300ms、主题切换逻辑全部与 `JsonToGo.vue` 一致

## 测试 `json-to-ts.test.js`

测试用例（参照 `json-to-go.test.js` 结构）：

1. 空输入返回 `{ ok: true, code: '' }`
2. 非法 JSON 返回 `{ ok: false }` 且 `error` 匹配 `/JSON/`
3. 平面对象生成 `interface AutoGenerated {`，包含 `string` / `number` / `boolean`
4. 嵌套对象生成多个 `interface` 声明
5. `allOptional: true` 时字段带 `?`（如 `name?:`）
6. `allOptional: false`（默认）时字段不带 `?`
7. 顶层是数组时正常生成
8. null 字段正常生成
9. 混合类型数组正常生成
10. 自定义 `typeName: 'User'` 生成 `interface User {`
11. 非法 `typeName` 回退 `AutoGenerated` 且返回 warning
12. 合法 `typeName` 无 warning
13. 空 `typeName` 无 warning，使用 `AutoGenerated`
14. `preferTypes: true` 输出 `type AutoGenerated = {` 而非 `interface`
15. `preferTypes: false`（默认）输出 `interface`

## 风险

- `quicktype-core` 的 TypeScript renderer 在 `allPropertiesOptional` 下的具体输出形态需实测确认（是 `field?: T` 还是 `field: T | undefined`）。如果实测发现不符合"所有属性可选"的预期，调整方案（如改用 `prefer-unknown` 或自定义后处理）。测试用例 #5 会锁定这一点。
- `@codemirror/lang-typescript` 的 `typescript()` 函数默认同时启用 TS + JS 解析；对纯 TS 输出足够。

## 实现顺序

1. 装 `@codemirror/lang-typescript`
2. 写 `json-to-ts.js` + `json-to-ts.test.js`，跑测试确认（重点验证 #5 和 #14）
3. 写 `JsonToTs.vue`
4. 接入 `router.js` / `routes.js` / `tools.js`
5. `npm run lint` + `npm run test`
